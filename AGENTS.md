## Project Overview

**Name**: verified-solidity
**Purpose**: Write a smart contract once, get formally verified Solidity out. An actively maintained, independent continuation of the Reach language and verifying compiler (Apache-2.0), with a modernized toolchain and a first-class verified-Solidity output mode: the compiler runs Z3-based formal verification on every compile and — only if verification succeeds — emits a self-contained `.sol` file, its ABI, and a machine-readable verification report for standard Ethereum tooling (Foundry, Hardhat, ethers, viem). Verified-or-absent by construction. The full Reach platform (JS runtime, Algorand backend, dockerized devnets, ~255 example DApps) remains in-tree and functional.
**Primary Language**: Haskell (stack/GHC 9.6.7, resolver lts-22.44) for the compiler and CLI; TypeScript for the JS runtime/standard library
**Framework**: None (compiler + language toolchain); legacy distribution is via Docker images (`reachsh/reach`, `reachsh/reach-cli`, `reachsh/runner`, etc.); the verified-Solidity path needs no Docker
**Repository Type**: Monorepo
**Current Version**: 0.1.13 (rc.3) — see `VERSION`
**Toolchain pins** (`DEPS`): solc 0.8.26, z3 4.12.5; stack v2.15+ required

## Architecture

- **Service type**: Compiler + CLI toolchain; users interact via `./reach` (or `scripts/reach-sol` / `reachc --sol` for the verified-Solidity path, which needs no Docker)
- **Compiler pipeline**: `.rsh` source → JS-like parse (`Parser.hs`) → evaluation (`Eval/`) → linearization (`Linearize.hs`) → end-point projection (`EPP.hs`) → verification (`Verify/` via Z3 SMT) → backends (`Backend/` emits JS; `Connector/` emits Solidity for EVM chains and TEAL for Algorand)
- **Verification**: Every compile runs formal verification with Z3 (`hs/smt2/runtime.smt2` is the SMT runtime theory) — token linearity, balance sufficiency, arithmetic overflow, assertion honesty under honest and dishonest participant models
- **Verified-Solidity mode** (`reach sol`): ETH-only compile that emits `index.main.sol` + `index.main.abi.json` + `index.main.verify.json` into `build/`, only when verification succeeds; on failure the compile exits nonzero with a counterexample witness and emits no Solidity
- **EVM support**: Generated Solidity is compiled with `solc` 0.8.26 (pinned in `DEPS`), with `evmVersion` pinned to `paris` in `hs/src/Reach/Connector/ETH_solc.hs` (solc ≥0.8.20's default shanghai target emits PUSH0, which pre-shanghai chains and the bundled devnet geth reject); Solidity runtime support in `hs/sol/` includes a vendored OpenZeppelin ERC20
- **Database**: None
- **External integrations**: Docker Hub (legacy image distribution), CircleCI + GitHub Actions (CI), devnets for Algorand (`scripts/devnet-algo`) and Ethereum (`scripts/devnet-eth`)
- **Authentication**: None (developer tool)
- **Open/closed split**: `hs/package.open.yaml` is the open-source package definition; a closed "everest" variant activates via `package.closed.yaml` and the `MEVEREST` stack flag when present
- **Deployment target**: Docker Hub images (legacy); docs deployed via GitHub Actions

## Development Commands

### Setup

```bash
# macOS (see README.md; exact versions pinned in DEPS — check hs/Dockerfile.reachc if versions matter)
brew tap ethereum/ethereum
brew install haskell-stack z3 solidity
curl -sSL https://git.io/get-mo -o mo && chmod +x mo && sudo mv mo /usr/local/bin/

# Required: stack v2.15+, z3 4.12.5, solc 0.8.26, mo, and `goal` (or symlink scripts/goal-devnet as `goal`)
# Docker daemon required for image builds, js/ builds, docs builds, running examples,
# and the goal shim used by the full Haskell test suite
# CAUTION: mo needs bash >= 4.4 (`brew install bash`); under macOS system bash 3.2 it
# crashes and truncates its target file to empty — see Known Issues below
```

### Validation

```bash
# Fast sanity check — generates package.yaml, Version.hs, sol/stdlib.sol (~seconds)
cd hs && make expand

# Build the compiler, CLI, and tests (long: cold stack build can exceed 30 min)
cd hs && make hs-build

# Run the Haskell test suite (golden tests against hs/t/; ~35-45 min; ALGO tests need `goal`)
cd hs && make hs-test
# Filter tests: HS_TEST_ARGS='-p <pattern>' make hs-test
# Accept new golden outputs after intentional changes (see accept-run hazard in Known Issues):
cd hs && make hs-test-accept

# Typecheck/build check without running tests
cd hs && make hs-check

# Format Haskell (ormolu fork built in hs/ormolu-tool/, formats in place — no check-only mode)
cd hs && make hs-format

# Shell script lint (requires shellcheck — installed)
make sh-lint

# Dockerfile lint (requires hadolint — installed; currently fails on pre-existing
# warnings in legacy examples/ and devnet Dockerfiles)
make docker-lint

# Search for version-string drift and TODO markers (requires ag — installed)
make check
```

### Run Locally

```bash
# Verified-Solidity mode: ETH-only compile emitting .sol + .abi.json + .verify.json (no Docker)
REACH_DOCKER=0 ./reach sol examples/verified-solidity/index.rsh
# equivalents: reachc --sol, scripts/reach-sol; distributable binary: cd hs && make reachc-dist

# Foundry deploy test against the emitted .sol (forge installed locally)
examples/verified-solidity/foundry-test/run.sh

# Compile any example with a locally built compiler (no Docker)
REACH_DOCKER=0 ./reach compile examples/argz/index.rsh

# Build all Docker images (the default target; requires Docker daemon)
make

# JS runtime images (requires Docker)
cd js && make build     # or `make b` to skip react-runner

# Docs site (requires Docker); serves at http://localhost:8080
cd docs && make build && make serve-up

# Interactive compiler development (ghcid watch loops)
cd hs && make watch-reachc     # or watch, watch-reach-cli
```

## Coding Rules

### Style

- **Haskell**: formatted with ormolu (`make hs-format`, tool built in `hs/ormolu-tool/`); language extensions are declared globally in `hs/package.open.yaml` (`LambdaCase`, `OverloadedStrings`, `RecordWildCards`, etc.) — don't add per-file pragmas for extensions already in the default set. `EverestUtil.hs` is excluded from formatting (mid-expression `#ifdef` branches can't parse under ormolu's CPP masking)
- **TypeScript** (`js/stdlib/ts/`): follow existing style; builds via per-directory Makefiles inside Docker
- **Shell scripts**: POSIX sh where possible; must pass `shellcheck` (`make sh-lint`)
- Use descriptive names; keep functions focused

### Patterns

- **Testing**: golden tests — compiler output is compared against checked-in expectations in `hs/t/` (`y/` compiles successfully, `n/` must fail with the recorded error). After intentional output changes, regenerate with `make hs-test-accept` and review the diff carefully
- **Versioning**: the single source of truth is `VERSION`; dependency/image versions live in `DEPS`. Never hardcode version strings elsewhere (`make check` greps for drift)
- **Error handling (compiler)**: user-facing compiler errors have error codes documented in the docs site; new errors need docs entries

### Do NOT

- Do not edit generated files: `hs/package.yaml`, `hs/src/Reach/Version.hs`, `hs/sol/stdlib.sol` (edit their sources and run `make expand`)
- Do not hand-edit golden test outputs in `hs/t/` — use `make hs-test-accept`
- Do not modify vendored OpenZeppelin sources under `hs/sol/openzeppelin-contracts-*`
- Do not bump versions in `DEPS`/`VERSION` casually — image tags, CI, and the changelog (`docs/src/changelog/index.md`) depend on them
- Do not commit credentials, API keys, or secrets (`.env` is untracked — keep it that way)

### Escalation Triggers

- Any change to verification soundness (`hs/src/Reach/Verify/`, `hs/smt2/`)

- Version/dependency bumps in `VERSION` or `DEPS` (e.g., solc/z3 upgrades)
- Golden-test churn that isn't clearly explained by the change
- Test failures that cannot be resolved in 2 attempts

## Additional Context

### Known Issues / Current State

- **mo + macOS bash 3.2**: stock `/usr/local/bin/mo` crashes under bash 3.2 (`MO_FUNCTION_CACHE_HIT[@]: unbound variable`), and because the Makefile uses `>` redirection the target file (`hs/src/Reach/Version.hs`) is left **empty** while make considers it up to date — `rm` the target before retrying. Fix: `brew install bash` (mo's `env bash` shebang then picks up bash 5; applied on the primary dev machine 2026-07-22), or upgrade mo / patch the two unguarded `"${MO_FUNCTION_CACHE_*[@]}"` loops (lines ~907/913)
- **Full `hs-test` needs `goal`**: ALGO golden tests shell out to `goal clerk compile`; without it ~450 tests fail. Use a Docker-backed goal shim that mounts the repo and TMPDIR at identical container paths **and** passes `-w "$PWD"` (the stock `scripts/goal-devnet` mount is insufficient for reachc's absolute paths)
- **Accept-run hazard**: if Docker/`goal` is down, `make hs-test-accept` silently corrupts `t/n` goldens (the Docker error text gets accepted as expected output). Always `docker info` first and `git diff hs/t | grep -i docker` after
- **Bytecode-size goldens**: `t/y/tooBig` and `t/y/gh-1183` embed bytecode sizes and drift whenever solc/codegen changes — expected churn, audit that diffs are size integers only
- Toolchain modernization is complete: resolver lts-22.44 (GHC 9.6.7), solc 0.8.26, z3 4.12.5, all pinned in `DEPS`/`hs/stack.yaml`; last full suite run: 812/812 PASS (see `.agent/baseline.md`)
- Full builds (Haskell, js/, docs) are slow and/or Docker-dependent; prefer `make expand`, targeted `HS_TEST_ARGS` test runs, and `REACH_DOCKER=0 ./reach compile` loops during development
