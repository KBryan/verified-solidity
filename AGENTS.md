## Project Overview

**Name**: reach-lang (Reach)
**Purpose**: Reach is a domain-specific language and verifying compiler for building safe decentralized applications (DApps). One `.rsh` source program compiles to smart contracts for Ethereum-like networks (Solidity/EVM) and Algorand (TEAL), plus a JavaScript frontend interface, with automatic formal verification of key safety properties via an SMT solver.
**Primary Language**: Haskell (built with stack/GHC) for the compiler and CLI; TypeScript for the JS runtime/standard library
**Framework**: None (compiler + language toolchain); distribution is via Docker images (`reachsh/reach`, `reachsh/reach-cli`, `reachsh/runner`, etc.)
**Repository Type**: Monorepo
**Current Version**: 0.1.13 (rc.3) — see `VERSION`

## Architecture

- **Service type**: Compiler + CLI toolchain, distributed as Docker images; users interact via the `./reach` shell script
- **Compiler pipeline**: `.rsh` source → JS-like parse (`Parser.hs`) → evaluation (`Eval/`) → linearization (`Linearize.hs`) → end-point projection (`EPP.hs`) → verification (`Verify/` via Z3 SMT) → backends (`Backend/` emits JS; `Connector/` emits Solidity for EVM chains and TEAL for Algorand)
- **Verification**: Every compile runs formal verification with Z3 (`hs/smt2/runtime.smt2` is the SMT runtime theory)
- **EVM support**: Generated Solidity is compiled with `solc` (version pinned in `DEPS`); Solidity runtime support in `hs/sol/` includes a vendored OpenZeppelin ERC20
- **Database**: None
- **External integrations**: Docker Hub (image distribution), CircleCI + GitHub Actions (CI), devnets for Algorand (`scripts/devnet-algo`) and Ethereum (`scripts/devnet-eth`)
- **Authentication**: None (developer tool)
- **Open/closed split**: `hs/package.open.yaml` is the open-source package definition; a closed "everest" variant activates via `package.closed.yaml` and the `MEVEREST` stack flag when present
- **Deployment target**: Docker Hub images; docs deployed via GitHub Actions

## Development Commands

### Setup

```bash
# macOS (see README.md; exact versions pinned in DEPS — check hs/Dockerfile.reachc if versions matter)
brew tap ethereum/ethereum
brew install haskell-stack z3 solidity
curl -sSL https://git.io/get-mo -o mo && chmod +x mo && sudo mv mo /usr/local/bin/

# Required: stack v2.7.5, z3, solc, mo, and `goal` (or symlink scripts/goal-devnet as `goal`)
# Docker daemon required for image builds, js/ builds, docs builds, and running examples
```

### Validation

```bash
# Fast sanity check — generates package.yaml, Version.hs, sol/stdlib.sol (~seconds)
cd hs && make expand

# Build the compiler, CLI, and tests (long: cold stack build can exceed 30 min)
cd hs && make hs-build

# Run the Haskell test suite (golden tests against hs/t/)
cd hs && make hs-test
# Filter tests: HS_TEST_ARGS='-p <pattern>' make hs-test
# Accept new golden outputs after intentional changes:
cd hs && make hs-test-accept

# Typecheck/build check without running tests
cd hs && make hs-check

# Format Haskell (ormolu, in place — no check-only mode)
cd hs && make hs-format

# Shell script lint (requires shellcheck)
make sh-lint

# Dockerfile lint (requires hadolint)
make docker-lint

# Search for version-string drift and TODO markers (requires ag/the_silver_searcher)
make check
```

### Run Locally

```bash
# Build all Docker images (the default target; requires Docker daemon)
make

# JS runtime images (requires Docker)
cd js && make build     # or `make b` to skip react-runner

# Compile an example with a locally built compiler (no Docker)
REACH_DOCKER=0 ./reach compile examples/argz/index.rsh

# Verified-Solidity mode: ETH-only compile emitting .sol + .abi.json + .verify.json
# (also: `reachc --sol`, `scripts/reach-sol`, `cd hs && make reachc-dist`)
REACH_DOCKER=0 ./reach sol examples/verified-solidity/index.rsh

# Docs site (requires Docker); serves at http://localhost:8080
cd docs && make build && make serve-up

# Interactive compiler development (ghcid watch loops)
cd hs && make watch-reachc     # or watch, watch-reach-cli
```

## Coding Rules

### Style

- **Haskell**: formatted with ormolu (`make hs-format`); language extensions are declared globally in `hs/package.open.yaml` (`LambdaCase`, `OverloadedStrings`, `RecordWildCards`, etc.) — don't add per-file pragmas for extensions already in the default set
- **TypeScript** (`js/stdlib/ts/`): follow existing style; builds via per-directory Makefiles inside Docker
- **Shell scripts**: POSIX sh where possible; must pass `shellcheck` (`make sh-lint`)
- Use descriptive names; keep functions focused

### Patterns

- **Testing**: golden tests — compiler output is compared against checked-in expectations in `hs/t/` (`y/` compiles successfully, `n/` must fail with the recorded error). After intentional output changes, regenerate with `make hs-test-accept` and review the diff carefully
- **Versioning**: the single source of truth is `VERSION`; dependency/image versions live in `DEPS`. Never hardcode version strings elsewhere (`make check` greps for drift)
- **Error handling (compiler)**: user-facing compiler errors have error codes documented in the docs site; new errors need docs entries

### Do NOT

- Do not edit generated files: `hs/package.yaml`, `hs/src/Reach/Version.hs`, `hs/sol/stdlib.sol` (edit their sources and run `make expand`)
- Do not hand-edit golden test outputs in `hs/t/` — use `make hs-test-accept`
- Do not modify vendored OpenZeppelin sources under `hs/sol/openzeppelin-contracts-*`
- Do not bump versions in `DEPS`/`VERSION` casually — image tags, CI, and the changelog (`docs/src/changelog/index.md`) depend on them
- Do not commit credentials, API keys, or secrets (`.env` is untracked — keep it that way)

## Risk Areas

| Path / Area | Risk Level | Notes |
|---|---|---|
| `hs/src/Reach/Connector/` | High | Generates on-chain code (Solidity/TEAL) — bugs become deployed financial contracts |
| `hs/src/Reach/Verify/`, `hs/smt2/` | High | Formal verification core — soundness bugs silently void safety guarantees |
| `hs/sol/`, `hs/rsh/stdlib.rsh` | High | Language/contract runtime stdlib shipped into every compiled DApp |
| `js/stdlib/` | High | Runtime that handles accounts, keys, and transaction signing |
| `hs/src/Reach/Eval/`, `EPP.hs`, `Linearize.hs` | Medium | Core compiler semantics — changes ripple into all backends |
| `VERSION`, `DEPS`, Dockerfiles | Medium | Release and reproducibility surface |
| `reach`, `hs/app/reach/` | Medium | User-facing CLI |
| `examples/`, `hs/t/` | Low | Test material (but golden-output churn needs review) |
| `docs/`, `svg/` | Low | Documentation only |

## Review Expectations

### All Changes

- [ ] `cd hs && make hs-test` passes (or golden diffs from `hs-test-accept` are reviewed and intentional)
- [ ] `make expand` still succeeds; no generated files edited by hand
- [ ] Haskell changes formatted with ormolu (`make hs-format`)
- [ ] No unrelated changes included

### Compiler / Backend Changes

- [ ] Both connectors considered (ETH and ALGO) — a change to shared IR affects both
- [ ] Verification still passes on affected examples (`./reach compile` runs verify by default)
- [ ] Relevant examples in `examples/` still compile
- [ ] New user-facing errors documented in `docs/`

### Release-Related Changes

- [ ] `VERSION` bump paired with a changelog entry in `docs/src/changelog/index.md`
- [ ] `DEPS` version bumps verified against `hs/Dockerfile.reachc`
